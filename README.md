# HealthPredict — AI-Powered Patient Health Prediction Application

A full-stack web application that manages patient blood test records and generates AI-driven health assessments using the Anthropic API.

---

## Features

- **CRUD Operations** — Create, view, update, and delete patient records
- **AI Health Assessment** — Automatically analyses Glucose, Haemoglobin, and Cholesterol values via the Anthropic API and generates a personalised health remark
- **Input Validation** — Email format, date-of-birth (no future dates), and numeric blood-test values are all validated on both client and server sides
- **Search** — Filter patients by name or email instantly
- **Colour-coded biomarker badges** — Values are highlighted Normal / Low / High at a glance
- **Persistent Storage** — SQLite database; no external DB setup required

---

## Tech Stack

| Layer | Technology |
|-------|-----------|
| Backend | Python 3, Flask |
| Frontend | HTML5, Bootstrap 5, Font Awesome 6 |
| Database | SQLite (via Python's built-in `sqlite3`) |
| AI / ML API | Anthropic API (Claude Haiku model) |
| Environment | python-dotenv |

### Why this stack?

- **Flask** is lightweight and ideal for a self-contained application — minimal boilerplate, easy routing, and Jinja2 templating out of the box.
- **Bootstrap 5** delivers a responsive, professional UI without requiring a JavaScript framework.
- **SQLite** requires zero configuration and is perfectly suited for a single-server application of this scope.
- **Anthropic API** provides reliable, medically-appropriate language generation for interpreting blood test results.

## Getting Started

### Prerequisites

- Python 3.9+
- An [Anthropic API key](https://console.anthropic.com/)

### Installation

```bash
# 1. Clone the repository
git clone https://github.com/your-username/health-prediction-app.git
cd health-prediction-app

# 2. Create and activate a virtual environment (recommended)
python -m venv venv
venv\Scripts\activate        # Windows
# source venv/bin/activate   # macOS / Linux

# 3. Install dependencies
pip install -r requirements.txt

# 4. Configure environment variables
copy .env.example .env       # Windows
# cp .env.example .env       # macOS / Linux
```

Open `.env` and fill in your values:

```
ANTHROPIC_API_KEY=sk-ant-...
SECRET_KEY=any-long-random-string
```

### Running the Application

```bash
python app.py
```

Visit **http://localhost:5000** in your browser.

## API Endpoints

| Method | Route | Description |
|--------|-------|-------------|
| GET | `/` | List all patients (with optional `?search=`) |
| GET | `/add` | Add patient form |
| POST | `/add` | Create patient + trigger AI assessment |
| GET | `/view/<id>` | Patient profile page |
| GET | `/edit/<id>` | Edit patient form |
| POST | `/edit/<id>` | Update patient + re-run AI assessment |
| POST | `/delete/<id>` | Delete patient |
| GET | `/api/patients` | JSON list of all patients |

## Data Validation Rules

| Field | Rule |
|-------|------|
| Full Name | Minimum 2 characters |
| Date of Birth | Must be a past date (not today or future) |
| Email | Must match standard email format |
| Glucose | Positive numeric value (mg/dL) |
| Haemoglobin | Positive numeric value (g/dL) |
| Cholesterol | Positive numeric value (mg/dL) |

---

## Reference Ranges Used for Colour Coding

| Biomarker | Low | Normal | High |
|-----------|-----|--------|------|
| Glucose (fasting) | < 70 mg/dL | 70 – 99 mg/dL | > 99 mg/dL |
| Haemoglobin | < 12 g/dL | 12 – 17.5 g/dL | > 17.5 g/dL |
| Cholesterol | — | < 200 mg/dL | ≥ 200 mg/dL |
